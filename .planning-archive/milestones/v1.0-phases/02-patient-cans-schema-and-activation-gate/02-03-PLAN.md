    ### cans-parser.test.ts

    - Valid minimal frontmatter → returns { frontmatter: {schema_version: '1.0', ...}, body: '' }
    - No --- delimiters → returns { frontmatter: null, body: fullContent }
    - Empty frontmatter ---\n--- → returns { frontmatter: null, error: 'frontmatter is empty or not an object' }
    - Malformed YAML (e.g. unbalanced quotes) → returns { frontmatter: null, error: 'YAML parse error: ...' }
    - CANS.md with body content after --- → body field contains the trimmed body
    - YAML array (not object) at top level → treated as not-an-object error

    ### cans-integrity.test.ts

    Uses tmp directory (mkdtemp) for file I/O in tests.

    computeHash():
    - Same input → same output (idempotent)
    - Different input → different output
    - Output is 64 chars, all lowercase hex

    verifyIntegrity():
    - No sidecar file → { valid: false, reason: 'no-sidecar' }
    - Sidecar exists, hash matches → { valid: true }
    - Sidecar exists, hash differs → { valid: false, reason: 'hash-mismatch', stored: string, computed: string }

    writeIntegritySidecar():
    - Creates .CANS.md.sha256 file in same dir
    - Content is the sha256 hex of the provided content
    - Calling verifyIntegrity after writeIntegritySidecar returns { valid: true }
