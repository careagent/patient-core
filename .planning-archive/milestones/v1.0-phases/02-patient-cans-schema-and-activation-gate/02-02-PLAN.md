  <behavior>
    Pipeline steps (in order — must short-circuit on first failure):
    1. Presence check: fs.readFile(cansPath) — ENOENT returns { active: false, reason: 'no-cans' } silently
    2. YAML parse: parseCANS(content) — parse error returns { active: false, reason: parseError }
    3. identity_type discriminator: frontmatter.identity_type !== 'patient' returns { active: false, reason: 'CANS.md identity_type is "X", expected "patient"' }
    4. SHA-256 integrity: verifyIntegrity(cansPath, content) — no-sidecar returns { active: false, reason: 'CANS.md has never been signed. Run patientagent resign...' }; hash-mismatch returns { active: false, reason: 'CANS.md integrity check failed. Run patientagent resign...' }
    5. TypeBox schema: Value.Check(CANSSchema, frontmatter) — fails return { active: false, reason: 'CANS.md has N validation error(s). Run patientagent validate for details.', errors: [{path, message}] }
    6. All pass: { active: true, document: frontmatter as CANSDocument, reason: 'activated' }

    Audit callbacks: call this._auditLog for schema validation failure (include errorCount), integrity failure (include computed hash), and successful activation (include identity_type). Do NOT audit the silent no-cans case.
    All audit callbacks wrapped in try/catch (AuditPipeline is still a Phase 3 stub).

    Cases with I/O:
    input: valid CANS.md with matching sidecar → output: { active: true, document: { schema_version, identity_type, consent_posture, health_literacy_level, ... } }
    input: no CANS.md file → output: { active: false, reason: 'no-cans' }
    input: CANS.md exists, no sidecar → output: { active: false, reason contains 'never been signed' }
    input: CANS.md exists, sidecar has wrong hash → output: { active: false, reason contains 'patientagent resign' }
    input: CANS.md with identity_type: provider → output: { active: false, reason contains 'provider' }
    input: CANS.md with missing required fields → output: { active: false, errors: Array }
    input: CANS.md with invalid YAML → output: { active: false, reason contains 'YAML parse error' }
  </behavior>
