**RED phase:** Create `test/unit/audit/writer.test.ts` with failing tests covering:

1. **Hash chain basics:**
   - `append()` writes entries as JSONL to the log file (after flush)
   - First entry has `prev_hash: null` (genesis)
   - Second entry's `prev_hash` equals SHA-256 of the first entry's JSON line
   - Multiple entries form a valid chain (each prev_hash = hash of previous line)

2. **Async buffering:**
   - `append()` is synchronous and does not immediately write to disk
   - `flush()` writes buffered entries to disk
   - After flush, file contains all appended entries as newline-delimited JSON
   - Hybrid trigger: entries flush when buffer reaches threshold (set a low threshold like 2 for testing)
   - Timer trigger: entries flush after interval (use `vi.useFakeTimers()` to test)
   - Concurrent flush guard: calling flush() while flushing does not duplicate entries

3. **Chain verification:**
   - `verifyChain()` returns `{ valid: true, entries: N }` for valid chain
   - `verifyChain()` returns `{ valid: false, brokenAt: M, error: '...' }` for tampered chain
   - `verifyChain()` returns `{ valid: true, entries: 0 }` for empty/missing file

4. **Crash recovery:**
   - `recoverLastHash()` at construction reads last line of existing log and recovers chain position
   - New writer on existing log continues the chain correctly (prev_hash of new entry = hash of last disk entry)

5. **Edge cases:**
   - `getLastHash()` returns null before any append, returns correct hash after append
   - `flush()` on empty buffer is a no-op
   - `dispose()` clears the flush timer

Test fixtures: Use `node:fs/promises mkdtemp` for temp directories, clean up in `afterEach`.

Replace the stub with a working implementation:

```
class AuditWriter {
  private lastHash: string | null = null;
  private buffer: string[] = [];
  private flushTimer: ReturnType<typeof setTimeout> | null = null;
  private flushing: boolean = false;
  private readonly logPath: string;

  static readonly FLUSH_INTERVAL_MS = 1000;
  static readonly FLUSH_THRESHOLD = 10;
```

Key implementation details:
- **Constructor:** Call `recoverLastHash()` synchronously to read last line from existing log file. Use `existsSync` + `readFileSync` (sync is OK at construction time only). Ensure `.careagent/` directory exists via `mkdirSync({ recursive: true })`.
- **`append(entry)`:** Enrich with `prev_hash: this.lastHash`, serialize to JSON with explicit field ordering (schema_version, timestamp, session_id, trace_id, action, action_state, actor, target, outcome, details, blocked_reason, blocking_layer, correlation_id, summary, prev_hash), compute SHA-256 of the line, update `this.lastHash`, push to buffer. If buffer length >= FLUSH_THRESHOLD, call `void this.flush()`. Otherwise `scheduleFlush()`.
- **`flush()`:** Async. Guard with `this.flushing` flag. Splice all entries from buffer, join with newlines + trailing newline, write via `appendFile(this.logPath, data)`. After write, check if more entries arrived during flush and schedule if so.
- **`scheduleFlush()`:** Set timer for FLUSH_INTERVAL_MS. `.unref()` the timer so it does not keep Node.js alive.
- **`verifyChain()`:** Read the entire file synchronously (`readFileSync`). Split by newlines, skip empty lines. Walk the chain: parse each line as JSON, verify `prev_hash` matches SHA-256 of the previous line (null for first). Return `{ valid, entries, brokenAt?, error? }`.
- **`getLastHash()`:** Return `this.lastHash`.
- **`dispose()`:** Clear the flush timer if active.
- **`recoverLastHash()`:** Private. Read file if exists, split by newlines, take last non-empty line, compute SHA-256 of it. Return null if no file or empty.

**IMPORTANT:** The `append()` method must use explicit property ordering when constructing the JSON to avoid non-deterministic serialization (Pitfall 2 from RESEARCH.md). Build the entry object with properties in a fixed order, do NOT use object spread.

**IMPORTANT:** The flush timer MUST be `.unref()`'d to prevent hanging tests (Pitfall 3 from RESEARCH.md).

**IMPORTANT:** Make `FLUSH_INTERVAL_MS` and `FLUSH_THRESHOLD` static readonly properties on the class so tests can reference them or the class can be subclassed for testing with lower thresholds. Alternatively, accept options in the constructor for testability.
